GIT - Tree History Storage Tool

GIT is a "directory content manager" designed to handle absolutely massive projects with speed and efficiency, and the release of the 2.6.12 (and later) versions of the Linux kernel as well as more and more other projects switching to it would indicate that it does this task well.

GIT falls into the category of distributed source code management tools, similar to Arch or Darcs (or, in the commercial world, BitKeeper). Every GIT working directory is a full-fledged repository with full revision tracking capabilities, not dependent on network access to a central server.

Highlights:

  • Speed of use, both for the project maintainer, and the end-users, is a key development principle.
  • Two, interchangeable, on-disk formats are used:
    • An efficient, packed format that saves space and network bandwidth.
    • An unpacked format, optimized for fast writes and incremental work.
  • The history is stored as a directed acyclic graph, making long-lived branches and repeated merging simple.
  • The Core GIT project considers itself to provide low-level "plumbing" and common ground facility for other projects, as well as to serve to arbitrate for compatibility between them. The projects built on top of the core GIT are referred to as "porcelain". Cogito, StGIT, (h)gct, gitk, qgit, and gitweb are all building upon the core GIT tools, and provide an easy to use interface to various pieces of its functionality.
  • Some other projects have taken the concepts from the core GIT project and are either porting an existing toolset to use the GIT tools, or reimplementing the concepts internally, to benefit from the performance improvements. This includes e.g. Darcs-git.

Download

To get a copy of GIT:

  • Daily snapshots are available at http://www.codemonkey.org.uk/projects/git-snapshots/git/ (thanks to Dave Jones).
  • Source tarballs and RPMs at http://www.kernel.org/pub/software/scm/git/.
  • Via GIT itself:
    Core GIT git clone http://www.kernel.org/pub/scm/git/git.git LOCALDIR
    Cogito cg-clone http://www.kernel.org/pub/scm/git/git.git
    StGIT stg clone http://www.kernel.org/pub/scm/git/git.git LOCALDIR
  • Alternatively, using the commands above with a rsync://rsync.kernel.org/pub/scm/git/git.git URL. Rsync is generally faster for the initial clone, and you can switch later by editing .git/branches/origin (or using cg-branch-chg of Cogito) and changing the URL.

Porcelains

SCM Interface layers

Cogito
Cogito is the most popular version control system on top of GIT. It aims at seamless user interface and ease of use, providing generally smoother user experience than the "raw" Core GIT itself and indeed many other version control systems.
StGIT
Stacked GIT provides a quilt-like patch management functionality in the GIT environment. You can easily manage your patches in the scope of GIT until they get merged upstream.
(h)gct
(h)gct is a GUI enabled commit tool. It has support for both GIT and Mercurial.

History Visualization

gitk
gitk is a simple TK GUI for browsing history of GIT repositories easily.
qgit
qgit is a QT GUI for browsing history of GIT repositories, similar to gitk.
gitweb
GITweb provides full-fledged web interface for GIT repositories.

Documentation

The GIT distribution contain a tutorial in the Documentation subdirectory. Additionally, the Kernel-Hacker's GIT Tutorial may be useful. (Thanks to Jeff Garzik for that document.)

Development

GIT was initially written by Linus Torvalds with help of a group of hackers around the 'net. It is currently maintained by Junio C Hamano.

The GIT development takes place on the GIT mailing list. To subscribe, send an email with just "subscribe git" in the body to majordomo@vger.kernel.org. Mailing list archives are available at http://marc.theaimsgroup.com/?l=git.
